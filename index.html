Enhancement One

Enhancement One: Software Design and Engineering

For this enhancement, Software Engineering, and Design, I chose to focus on my first artifact, possibly the one I will be working on through this course. This artifact is an android eCommerce app to sell location coffee shop products. I select this artifact because mobile application development is a great skill to have as a developer and in demand. The second reason I choose this artifact is that it embodies various capabilities needed in the tech industry. Lastly, it showcases my experience with multiple tools such as Firebase API, Facebook API, and the SQLite database embedded in the android mobile OS. I added a second authentication method, which uses firebase because some users might not have Facebook. I will be transitioning from the SQLite database to the firebase database. The reason for that is to add flexibility so that users can instantly access new products without having to reinstall the app.

I have not fully met the objectives I planned for this enhancement yet because I believe there are still some code structures and designs that need improvement.

In this process, the lesson learned is that some API, such as Facebook, could be inflexible when implementing it along with other API. I also learned that following the standard design, best practices would save ample time in the long run. The challenges I faced were finding detailed information on how to implement various APIs. As this course progresses, I will keep this design principle from opensource.com in mind “globals are bad. Functions are better than types. Objects are likely to be better than complex data structures” (30 best practices for software development and testing).

Enhancement Two

Enhancement Two: Algorithms and Data Structure

As mentioned in the TutorialPoing, Data Structures are the programmatic way of storing data so that data can be used efficiently (Data Structure and Algorithms Tutorial). In this assignment, I decided to enhance another part of the Android mobile app, LCS2(Local Coffee Shop), a simple eCommerce app that displays some random items when it starts and gives users the option to search for products, add to cart and place an order. I used the firebase Real-time database to facilitate the implementation of product search, storing, and retrieving order history data. The firebase API method helps with the application of the search algorithm that I used to display products, based on user-entered character(s). According to the Tutorial point site, Algorithm is a step-by-step method, which determines a set of directions to be performed in a specific order to get the desired output (Data Structure and Algorithms Tutorial). I redesign the Order class to include the list of items in the Order, orderDate, total, and ID used in the OrderHistoryAdapter class to display a list of orders.

I structure the order collection based on user ID and date keys. This method, to my knowledge, is better and more straightforward compare to using SQL, where data distributed between multiple tables.

The lesson I learned is that exploring a tool before implementing it is a must. I assumed firebase would have a similar querying syntax as in MongoDB, which was not correct. I also discovered that the Firebase Real-time database is for unstructured data, whereas the Cloud Firestore is for structure data.

Enhancement Three

Enhancement Three: Databases

For this milestone, I select the zookeeper artifact that I worked on in IT-145 using java. The project was a simple role-based authentication system that takes in the zookeeper name and password then determines if he/she is a zookeeper, veterinarian, or administrator. All the credentials retrieved from a file called credentials and using "while/for-loop" to loop through each line of the text file and extract the password and role. The users had only three attempts to authenticate; otherwise, the program will terminate successfully.

Enhancement:

I chose to move the credentials to a MySQL database that is more secure and provides more flexibility than a text file. I also decided to move the program from the command line interface to a web interface that is user-friendly and easily accessible across multiple platforms.

I selected this artifact and the enhancement because it shows my ability to work web development projects using java server faces, Html, and CSS. I did meet the course objectives I planned to comply with this enhancement, which was implementing a database in a server-side project.

I learned that keeping up to date with all the new versions of API and technology is a must for software engineers. Also, I learned that authentication for the mobile app is a lot simpler compare to web applications.

This artifact can be view at www.cour2web.com

Educational Journey and Program Reflection

I have been fascinated by technologies since I was little, especially electronics and anything to do with science. I took my first programming class using Java from September to December 2007, a pre-requisite for AS in mechanical engineering. From there, I decided to get a Certificate as a Java Server Side Programmer. I did not work in computer science because I always felt I needed to learn more, so in 2018, I started a Bachelor of Software Engineering program at SNHU. While in the program, I learned that understanding the concept of programming empowers a developer to learn new languages quickly. My understanding of programming and data structure has improved drastically. In CS-200, I learned some valuable programming skills using Python, which also applies to many other programming languages. I learned that developing a flowchart or pseudocode is essential in designing well organized and quality products. In CS-200, I understand what algorithm means in computer science and the importance of adequately naming variables and functions, as well as comments within the code that help the developer and his team when testing or debugging.

Developing an interactive 3D object from scratch using C++ and OpenGL was an exciting experience, partly because I hadn't heard of OpenGL before.
